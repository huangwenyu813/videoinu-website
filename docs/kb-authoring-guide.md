# Knowledge Base Authoring Guide

## Goal
Write documents that are easy for both users and retrieval systems to use.

## Required sections
- Summary
- When to use
- Steps
- Tips
- Limitations
- Related
- Keywords

## Writing rules
- Prefer short steps over long prose
- Keep product names and feature names consistent
- Avoid vague claims
- If a workflow is conditional, say when it applies

## Status rules
- draft: not retrievable
- review: not retrievable
- published: retrievable
- deprecated: archived, not retrievable
